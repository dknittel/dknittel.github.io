Javascript Objects

Javascript objects are data structures that contain properties. There are three ways to create an entry into an object. The first is by putting it directly in the object such as:

var justin = { age: 21 }

You can also do it outside of the object with either of the following code (assuming the friends object has already been created):

justin[age] = 21

justin.age = 21

You can put a function within an object and call it. Here's an example:

justin.greeting = function() { console.log("hello") } justin.greeting(); //prints "hello"

Javascript objects are visually very similar to ruby hashes but there are a few key differences which we will see now.

Ruby Hashes

Unlike Javascript objects which are collections of properties that point to other objects, ruby objects contain a hash of instance variables that refer to different objects. In Ruby, instance variables can only be accessed inside of the context of an object in a method call. In addition, hash values can only be created within the hash or through code like this:

justin[age] = 21

Elements within a hash are referred to as key-value pairs. These key value pairs can be defined either with symbols or with the "rocket ship" notaion of =>. Here are examples of each:

justin = { age => 21 }

justin = { age: 21 }

Using the second style makes age a symbol, which can be accessed like this:

justin[:age]

Each language also has differnet built-in methods to access items in the hash or object. For example, each_key in Ruby allows you to loop through each key in the hash, passing the block to all of the keys. The delete function allows you to delete the key-value pair whose key matches the parameter of the function. has_key?(key) lets you know if the given hash has the key that is in the parameter.

Javascript also has useful functions for its objects. For example, Object.create() creates a new object using the object and properties passed into the function. Object.getOwnPropertyNames() returns an array that has each of the names of the object's enumerable and non-enumerable properties.

Javascript objects and ruby hashes can both be used to store lists of entries with two elements for each entry. The primary differences involve how you define and access these elements.
